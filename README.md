# 🧪 Manual Testing Project - SauceDemo

## 📌 Objective
To perform end-to-end manual testing on a web-based e-commerce application to ensure functionality, usability, and reliability.

## 🌐 Application Under Test
https://www.saucedemo.com/

## 🧩 Modules Covered
- Login Page
- Inventory (Product Listing) Page
- Product Details Page
- Cart Page
- Checkout Page

## 🔍 Testing Types Performed
- Functional Testing
- UI Testing
- Regression Testing
- Negative Testing
- Exploratory Testing

## 📝 Test Deliverables
- ✔️ Test Cases (60+ scenarios covering all modules)
- ✔️ Bug Reports (identified issues with severity & priority)
- ✔️ Test Plan (scope, strategy, and execution details)

## 🐞 Sample Bugs Identified
- Incorrect error message behavior on invalid login
- UI misalignment in product page (if any)
- Cart update inconsistencies (example)

## 📊 Tools Used
- Google Sheets / Excel (Test Cases & Bug Reports)

## 🚀 Execution Summary
All test cases were executed manually. Both positive and negative scenarios were validated to ensure application stability.

## 📌 Conclusion
The application was tested across major modules, and defects were identified and documented. Overall functionality works as expected with minor improvements suggested.
